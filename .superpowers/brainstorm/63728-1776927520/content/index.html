URL Routing Structure (SPA)
GET /login
  — Domain Authentication (Task 2)
GET /dashboard
  — "Active Queries" list & performance stats
GET /query/:id
  — The 3-Pane Workspace (Task 3-5)
GET /settings
  — API Keys & User profile

SPA Architecture Design:

  • Client-Side Routing: Using React Router for deep-linking.
  • State Persistence: Query state stays in memory during session, allowing fast switching between "Active Queries".
  • Deep Linking: Direct access to specific investigations (e.g., /query/742).
