Deployment Strategy: Docker vs. Bare Metal

Deciding how to host the Centralized Security Hub on your dedicated virtual server

A

Docker Compose (Planned)

Self-contained units. Easy to manage PostgreSQL, FastAPI, and Node.js dependencies without polluting the host OS. Portable and reproducible.

B

Bare Metal / Systemd

Run FastAPI and PostgreSQL directly on the host OS. Lower overhead, simpler networking for SSH/LDAP, but requires manual OS-level dependency management (Node, Python versions).

Key Question for you:

Since we're on a dedicated virtual server, do you have a preference for keeping the host "clean" (Docker) or do you prefer the direct access and performance of running services directly on the OS?
