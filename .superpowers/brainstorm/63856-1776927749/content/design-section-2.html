Design Section 2: Streaming Communication (SSE)

Data Flow: Hub → Web Portal
1. Client Request: POST /api/orchestrate (JWT Auth)
↓
2. Hub Processing: Agent starts "Thinking" & tool execution.
↓
3. SSE Stream: Hub pushes chunks to browser via text/event-stream.
event: thought
data: {"text": "Searching VirusTotal..."}

event: message
data: {"text": "Found 3 malicious flags..."}

Implementation Details:

  • Protocol: Server-Sent Events (SSE). It's more efficient than WebSockets for one-way agent → user streaming and works natively with FastAPI's StreamingResponse.
  • Frontend Consumption: Use the EventSource API or a library like fetch-event-source (to support POST requests with headers) to update the Zustand store.
  • State Sync: Each chunk is appended to the message history in real-time, triggering React re-renders for the "live" typing effect.
