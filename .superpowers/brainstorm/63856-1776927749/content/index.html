Design Section 1: Frontend Architecture & Tab Persistence

Client-Side Tech Stack
  • React (Vite): Fast development and optimized production builds.
  • Zustand: Lightweight state management to handle active tabs and streaming content across the SPA.
  • React Router: Maps URLs like /query/:id to the correct tab in the workspace.

Persistent Tab Logic:

To ensure tabs persist across refreshes, we will implement the following flow:

  1. Database: Add a user_active_queries table to PostgreSQL (user_id, query_id, tab_order, is_active).
  2. Initial Load: Upon logging in, the React app fetches the list of active queries from /api/users/me/tabs.
  3. Sync: Opening, closing, or reordering a tab triggers a background POST/DELETE to the backend to keep the database in sync with the UI.
