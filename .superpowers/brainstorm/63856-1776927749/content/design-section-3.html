Design Section 3: Automated Evidence Extraction

Evidence Recognition Pipeline
Raw Agent Output: "I've checked the IP 192.168.1.50 and it is communicating with malware-c2.com."
Regex + LLM Extraction
Automated Evidence Locker
IP: 192.168.1.50 Domain: malware-c2.com

Automated Population Logic:

  1. Client-Side Parsing: As the stream chunks arrive, the React frontend runs a background parser (Regex-based) to identify indicators (IPv4, Domains, SHA256).
  2. Uniqueness Check: New indicators are added to the query's Evidence Locker state only if they aren't already present.
  3. Persistence: When the "Active Query" is closed or auto-saved, the evidence list is synced to the query_evidence database table.
